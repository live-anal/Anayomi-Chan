--------------------------------------------------------------------------------
　棒読みちゃん  Ver0.1.11.0 Beta16
--------------------------------------------------------------------------------
■概要
　・漢字を含む日本語の文章を音声合成で読み上げるツール。
　
　※アクエスト社の規則音声合成ライブラリ「AquesTalk」を利用しています(DLL同梱)
　　AquesTalkフォルダの配下にあるDLL等の著作権は同社に帰属します。
　　http://www.a-quest.com/
　　
　　【注意】
　　当ソフトが利用しているのは、旧版のAquesTalk(Win用)です。
　　最新版のAquesTalkとは利用規約が異なります。
　　旧版のAquesTalk(Win用)は、営利・非営利を問わず無償で利用できます。
　　詳しくは、AquesTalkフォルダ配下にあるAqLicense.txtをご覧下さい。

■主な機能
　・入力した文章の読み上げ
　　入力した日本語の文章を音声合成で読み上げます。
　　ドラッグ&ドロップしたTEXTファイルを読み上げます。
　　コマンドラインから読み上げの指示を送信できます。(ネット経由可) 
　
　・クリップボードの読み上げ
　　クリップボードを監視して、コピーされた文章を読み上げます。
　　WebブラウザやTextエディタなどでコピー操作(CTRL+C等)を行うと、
　　クリップボードにコピーされた文章を棒読みちゃんが読み上げます。
　
　・２ちゃんねる専用ブラウザの読み上げ
　　ログファイルの変化を検出して、新しいレスを読み上げます。
　　対応できてそうなソフトは以下の通りです。
　　　Open Jane / Jane Style / Jane View
　　　Live2ch / A Bone2 / ギコナビ / ホットゾヌ2
　　　注）2009/04/28時点での最新版でのみ確認 
　
　・Skypeの読み上げ（β）
　　Skypeから情報を取得して読み上げます。
　　チャットの文章、コンタクトのステータス変化、ムードメッセージの変化、
　　誰からの着信か、などの読み上げに対応しています。
　
　・Twitterの読み上げ（β）
　　タイムラインに流れてくる新着ツイートを読み上げます。
　　ストリーミングAPIでのリアルタイム取得にも対応しています。
　
　・ニコニコ実況・ニコニコ生放送の読み上げ（β）
　　ニコニコ実況・ニコニコ生放送のコメントを取得して読み上げます。
　　参加しているコミュニティの番組開始アラート機能もあります。
　
　・声質の切り替え機能
　　８種類の声質を標準で搭載しています。（AquesTalkのDLLを同梱）
　　さらに、SAPI5とSpeech Platformの音声合成エンジンも利用可能です。
　
　・辞書機能
　　単語をどのように発音するのか指定できます。
　　単語の読みを登録する辞書と、発音を修正する辞書があります。
　　正規表現での指定も可能です。 
　
　・タグ機能（ライブ配信者向け）
　　配信参加者がチャットログを通して棒読みちゃんに命令できます。
　　　声質/速度/音程/音量/残響の指定
　　　音声ファイルの再生
　　　コマンドの実行
　　　単語の読み方の教育/忘却（単純置換のみ）
　
　・アプリケーション連携機能（開発者向け）
　　他のアプリケーションからの指示で読み上げます。
　　Socket通信 か IpcClientChannel で接続できます。サンプルPG同梱。
　　また、プラグインを作成すれば棒読みちゃん本体への機能追加も可能です。

■解説
　このソフトは、入力された漢字混じりの日本語の文章を、
　IMEの再変換を利用して、かな文字だけの文章に変換します。
　そして、そのかな文字だけの文章をチョチョイと弄って音声記号列を生成し、
　アクエスト社がフリーで提供して下さっている「AquesTalk」で音声にします。
　要するに、ほとんど他力本願なソフトです（＾＾；
　
　AquesTalk自体は、すばらしい発音が可能なのですが、
　IMEの再変換では、漢字をどのように発音するかという情報が得られないため、
　どうしても、棒読みちゃんになってしまいます・・
　辞書を登録して、日本語を教えてあげてください！

■インストール方法
　・適当なフォルダに、ZIPファイル内の全てのファイル展開するだけです。
　　※ZIPフォルダから直接実行はできません。必ずどこかに展開してください。
　
　・旧バージョンからの更新時も、ファイルを上書きするだけですが、
　　念のためバックアップを取ってから行ってください。（特に辞書ファイル）
　　※更新時は、ユーザが育てた辞書と、添付の辞書がマージされます。
　　　重複する項目は、ユーザが育てた辞書が優先されますので、
　　　基本的には、それほど問題ないと思うのですが、
　　　過去に削除した項目が復活してしまう場合がありますのでご注意ください。
　　　更新前の辞書は、DicBakフォルダ配下ににバックアップされます。

■アンインストール方法
　・インストールしたフォルダ内のファイルを削除してください。
　※レジストリは利用していません。
　※「%APPDATA%\Fight Not Found\BouyomiChan」に設定の一部が保存されています。
　　気になる方はこちらも削除してください。

■使い方
　※画面上の項目にマウスを合わせるとヒントが表示されます
　　
　・起動方法
　　「BouyomiChan.exe」を実行してください。（唇のアイコン）
　　※起動しない場合は、前述の「事前準備」を行ってください。
　　※今まで動いていたのにある日突然起動しなくなった場合は、
　　　設定ファイルが壊れてしまった可能性があります。
　　　棒読みちゃんをインストールしたフォルダ内にある、
　　　「BouyomiChan.setting」や「Plugin_○○.setting」を、
　　　削除するか別のフォルダに移動してから起動してみてください。

　・入力した文章を音声合成する方法
　　1)音声合成画面の中央にあるテキストボックスに文章を入力する。
　　2)「声質」や「音量」や「速度」や「音程」を選択する。
　　3)画面左上の「音声合成」ボタンを押す。
　　※テキストファイルをドラッグ＆ドロップで読み込むこともできます。

　・辞書登録の方法
　　「辞書登録」タブから、単語や発音を登録できます。
　　辞書には以下の４種類があります。
　　・単語  … 単語の読み方を指定します（単純置換/正規表現）
　　・発音  … 発音を修正します（単純置換/正規表現）
　　・教育  … Studyタグで登録された単語が登録されていきます（単純置換のみ）
　　・タグ  … 特定の文字列をタグに変換します（単純置換/正規表現）
　　※発音辞書はAquesTalkの場合だけ有効です。SAPI5対応エンジン等では無視します。
　　※「その他→変換の順序について」の説明もご参照ください。
　　例）「tel」を「でんわ」と読ませたい場合
　　　1)「辞書登録→単語」タブを開きます。
　　　2)「探索文字列」欄に「tel」と入力します。※TELと大文字に統一されます。
　　　4)「置換後」欄に「でんわ」と入力します。
　　　5)「新規追加」ボタンを押します。

　・クリップボード監視の使い方
　　1)プラグインを有効にします。（その他→プラグイン）
　　2)他のソフト側でテキストのコピー操作(CTRL+Cなど)を行います。
　　3)棒読みちゃん側が自動的にコピーされたテキストの読み上げを開始します。
　　※音声合成タブのアイコンで監視の有効/無効を切り替えできます。

　・2ch専用ブラウザ読み上げの使い方
　　1)プラグインを有効にします。（その他→プラグイン）
　　2)2ch専用ブラウザ側で、スレッドを読み込みます。
　　3)棒読みちゃん側が自動的に増分のレスの読み上げを開始します。
　　※音声合成タブのアイコンで監視の有効/無効を切り替えできます。
　　※初回起動時に2ch専用ブラウザのインストール先を自動検索しますが、
　　　「Program Files」の直下以外にインストールしている場合は、
　　　手動でインストール先（ログの格納場所）を指定する必要があります。
　　　「その他→プラグイン」からプラグインの設定を行って下さい。

　・Skype読み上げの使い方
　　1)プラグインを有効にします。（その他→プラグイン）
　　2)Skypeを起動します。（Skypeを先に起動していても問題ありません）
　　　初回接続時には、Skypeの側で認証作業が必要です。
　　　Skype側に認証画面が表示されますので、
　　　「BouyomiChan.exe」がアクセスすることを許可してください。
　　3)Skypeを普段通り使用します。
　　　Skypeでのチャットの文章や、ステータス変化などを、
　　　棒読みちゃんが自動的に読み上げます。
　　※音声合成タブのアイコンで読み上げの有効/無効を切り替えできます。

　・Twitter読み上げの使い方
　　1)プラグインを有効にします。（その他→プラグイン）
　　2)利用許可の設定画面が表示されます。(初回起動時のみ)
　　　画面の指示に従って操作してください。以下のような流れになります。
　　　- ブラウザが開きTwitterのOAuth許可画面が表示される。
　　　- ブラウザ上でアカウント名とパスワードを入力して「許可」ボタンを押す。
　　　- ブラウザ上に暗証番号が表示される。
　　　- 棒読みちゃんの暗証番号入力画面に暗証番号を入力する。
　　3)取得対象の取り込み(初回起動時のみ)
　　　フォローしているユーザーやリストを取得対象として取り込みます。
　　　取り込みには時間がかかりますので、ゆっくりしていってね！
　　　また混雑している時間帯などは失敗する場合があります・・
　　4)設定に従ってつぶやきの取得が行われます。
　　※音声合成画面のTwitterアイコンを押すとTwitter画面が表示されます。
　　※Streaming APIは１ユーザにつき１コネクションまでなので、
　　　他のソフトでもStreaming APIを利用している場合はどちらかが切れます。

　・ニコニコ読み上げの使い方-①コメントの読み上げ
　　※IE系ブラウザでのみ利用できます(IEのCookieを利用して接続します)
　　1)プラグインを有効にします。（その他→プラグイン）
　　2)音声合成画面のニコニコアイコンを押してコメント取得画面を開きます。
　　3)コメント取得画面のURL欄に番組URLを入力し、ENTERキーを押します。
　　　番組URLの例：http://live.nicovideo.jp/watch/lv9999999
　　4)コメント読み上げが始まります。
　　※IE系ブラウザでログインしていない場合は、
　　　内蔵ブラウザが開きますので、ニコニコ生放送にログインしてください。
　　　IE系以外のブラウザ(FirefoxやChrome等)でログインしている場合、
　　　そちらの接続は切れてしまいますのでご注意ください。
　　
　・ニコニコ読み上げの使い方-②番組開始アラート
　　1)プラグインを有効にします。（その他→プラグイン）
　　2)プラグインの設定画面からメールアドレスとパスワードを設定します。
　　3)音声合成画面のニコニコアイコンを押してコメント取得画面を開きます。
　　4)コメント取得画面のアラートアイコンをクリックします。
　　5)参加コミュニティの番組が始まると通知されます。
　　※参加していないコミュニティもお気に入りとして手動登録できます。

　・タグ機能の利用法
　　1)タグ機能の有効化
　　　画面上の「配信者向け機能」ボタン(TVマーク)を有効にしてください。
　　　また、一部のタグはデフォルトでは無効になっています。
　　　設定画面→配信者向け機能→タグ より設定してください。
　　2)タグ辞書の登録
　　　配信参加者がチャットログにタグを直接記入しても何も起こりません。
　　　事前に配信者側で特定の文字列がタグに置換されるように、
　　　タグ辞書に登録しておく必要があります。
　　　辞書登録→タグ の画面から登録できます。
　　※タグ辞書はデフォルトで以下のような規則が登録されてます。かっコマンド風。
　　　y)ゆっくりです     → (Voice 1)(Speed 60)ゆっくりです
　　　b)女性２です       → (Voice 2)女性２です
　　　h)男性１です       → (Voice 3)男性１です
　　　d)男性２です       → (Voice 4)男性２です
　　　a)中性です         → (Voice 5)中性です
　　　r)ロボットです     → (Voice 6)ロボットです
　　　t)機械１です       → (Voice 7)機械１です
　　　g)機械２です       → (Voice 8)機械２です
　　　やまびこ)やふー    → (Reverb 2.5 0.7 40)やふー
　　　エコー)うわああ    → (Reverb 0.5 0.1 40)うわああ
　　　右）みぎから       → (Volume 0 100)みぎから
　　　左）ひだりから     → (Volume 100 0)ひだりから
　　　両）いつもどおり   → (Volume 100 100)いつもどおり
　　　声(1)あああ        → (Voice 1)あああ
　　　速度(200)いいい    → (Speed 200)いいい
　　　音程(150)ううう    → (Tone 150)ううう
　　　音量(50)えええ     → (Volume 50)えええ
　　　音量(0 100)        → (Volume 0 100)みぎからー
　　　残響(3.0 0.2 60)お → (Reverb 3.0 0.2 60)お
　　　教育(単語=よみ)    → (Study 単語=よみ)
　　　忘却(単語)         → (Forget 単語)
　　　再生(xx)           → (Sound xx) ※xx.wav/xx.mp3/xx.wmaが存在すれば再生
　　※タグの種類は以下の通りです。
　　　(Voice    value)    … 声質(1～8:AquesTalk,10001～:SAPI)
　　　(Speed    value)    … 速度(50:半分、100:標準、200:二倍)
　　　(Tone     value)    … 音程(50:半分、100:標準、200:二倍)
　　　(Volume   value)    … 音量(50:半分、100:標準、200:二倍)
　　　(Volume   l r)      … 音量(l:左の音量、r:右の音量)
　　　(Reverb   a b c)    … 残響(a:余韻時間、b:反復間隔、c:増幅率)
　　　(Sound    file)     … 音声ファイルの再生
　　　(SoundW   file)     … 音声ファイルの再生（再生終了まで待機）
　　　(Command  file)     … コマンドの実行
　　　(CommandW file)     … コマンドの実行（実行終了まで待機）
　　　(Study    word=read)… 単語の教育（教育辞書に登録されます）
　　　(Forget   word)     … 単語の削除（教育辞書の単語のみ削除可）
　　　※Voice/Speed/Tone/Volume/Reverbの効果は同じ行に限られます。
　　　※Speed/Tone/Volumeの引数は画面上の設定に対するパーセンテージです。
　　　※Sound/SoundWの引数のファイル名は拡張子を省略できます。
　　　　省略した場合はWAV,MP3,WMAの中で最初に見つけたファイルを再生します。

　　マウスの代わりにキーボードでも操作できます。
　　・音量　　　　　　　　　：↑↓

　　・全ての行をキャンセル　：Delete

　　・コンパクトモード　　　：F11
　　・設定画面表示　　　　　：F10
　　・配信者向け機能切り替え：F9
　　・簡易ブラウザ表示　　　：F8
　　・入力欄クリア          ：F7
　　※入力フォーカスの位置によっては機能しないことがあります(^^;

　・アイコンの変更方法

　　System\BouyomiChan.icoをお好みの画像ファイルに差し替えてください。

　・RemoteTalk.exeの利用法

　　Socket通信を利用していますので、同一PC上に限らず、
　　ネットワークを介して別のPCから指示を送ることもできます。たぶん。
　　詳細は、RemoteTalk.txtをご覧下さい。

　・他のアプリケーションからの利用法
　　Socket通信 又は IpcClientChannel を利用して、
　　他のソフトから本ソフトへ、読み上げの指示を送る事ができます。
　　また、プラグインを作成して棒読みちゃんに機能を追加することもできます。
　　詳細は、「SampleSrc」フォルダ配下のサンプルソースコードをご参照下さい。
　　サンプルソースコードは、自由に改変・利用して頂いて構いません。

■その他
　・音声記号列について
　　音声記号列とは、どのように発音するかを表す文字列です。
　　発音辞書での置換後の文字列は、音声記号列の仕様に則った形式にして下さい。
　　また、単語辞書での置換後の文字列も音声記号列で記入することができます。
　　音声記号列には、アクセント[']、区切り[/;+,、。？] などなどがあります・・
　　詳細はアクエスト社の資料をご覧ください。
　　http://www.a-quest.com/download/manual/siyo_onseikigou.pdf
　　※音声記号列はAquesTalk(デフォルトの８つの声質)の場合のみ有効です。
　　　SAPI5対応エンジン利用時の単語置換で音声記号列が含まれていた場合には、
　　　音声記号列を削除したうえで置換します。

　・変換の順序について
　　棒読みちゃんでは、入力された文章を下記の順序で変換していき、
　　AquesTalkが扱える音声記号列を生成します。
　　辞書登録の際には、直前のステップの文字列に合致するように登録して下さい。
　　01)文字種統一          … 辞書登録の重複を減らすため文字の種類を統一する。
　　02)タグ置換（正規表現）… 特定の文字列をタグに置換する。  （ユーザ辞書）
　　03)タグ置換            … 特定の文字列をタグに置換する。  （ユーザ辞書）
　　04)教育置換（正規表現）… 参加者が教育した単語を置換する。（ユーザ辞書）
　　05)教育置換            … 参加者が教育した単語を置換する。（ユーザ辞書）
　　06)単語置換（正規表現）… 登録した単語を置換する。        （ユーザ辞書）
　　07)単語置換            … 登録した単語を置換する。        （ユーザ辞書）
　　08)文節分解・漢字逆変換… 漢字を平仮名に変換する。
　　09)発音置換（正規表現）… 発音を置換する。                （ユーザ辞書）
　　10)発音置換            … 発音を置換する。                （ユーザ辞書）
　　11)音声記号列変換      … AquesTalkで扱える形式に変換する。
　　※SAPI5対応エンジン利用時には、08～11は行いません。

　・前処理置換について
　　前処理置換は、上記の変換順序の「01)文字種統一」の直前に実行される置換です。
　　通常の単語置換は文字種統一後に行われる為、「a」と「A」を区別できませんが、
　　前処理置換であれば区別できます。
　　必要な場合は、ReplacePre.dic/ReplacePreRegex.dicを直接編集してください。

　・後処理置換について
　　後処理置換は、上記の変換順序の「11)音声記号列変換」の直後に実行されます。
　　音声記号列を整形したい場合に利用できます。
　　必要な場合は、ReplacePost.dic/ReplacePostRegex.dicを直接編集してください。

　・「は」と「へ」の扱いについて
　　文章内の「は」を「わ」に、「へ」を「え」に自動的に変換する場合があります。
　　間違った発音になってしまう場合は、辞書で対応して下さい。
　　また、辞書に登録した読み方が間違って変換されてしまう場合は、
　　読み方をカタカナで登録してみてください。
　　それでも無理な場合はあきらめてください（＾＾；

　・英字の扱いについて
　　英字が1文字の場合はアルファベットの読みとなります（A   →エー）
　　英字が1文字より長い場合はローマ字っぽく読みます　（AIU →アイウ）
　　ローマ字として解釈できない場合は適当に読みます 　（TALK→タルク）
　　上記の規則で間違った読みとなる単語は、辞書で対応して下さい。
　　※デフォルトの境界は1文字ですが、設定で変更できます。

　・漢字の変換について
　　漢字の変換はIMEの再変換を利用しています。
　　再変換では、単純にひらがなへの変換を行うだけなので、
　　発音に関する情報は得られないため棒読みになりがちです。
　　漢字の変換が間違っている場合や、発音がおかしい場合は辞書で対応して下さい。
　　※SAPI5対応エンジン利用時には、棒読みちゃん側での漢字変換は行いません。

　・Microsoft Speech Platformの利用について
　　Speech Platformは、音声応答サーバー向けに提供されているようですので、
　　棒読みちゃんを利用した音声応答サーバの開発・運用の目的でのみご利用下さい。

　・生成した音声の取扱について
　　声質「女性1/女性2/男性1/男性2/中性/ロボット/機械1/機械2」については、
　　アクエスト社のAquesTalk(旧ライセンス版)を利用して生成していますので、
　　営利・非営利を問わず無償で利用できます。
　　詳しくは、AquesTalkフォルダ配下にあるAqLicense.txtをご覧下さい。
　　それ以外の声質については、音声合成エンジン毎にライセンスが異なります。
　　それぞれの音声合成エンジンの製造元にご確認ください。
